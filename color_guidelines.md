## 4. 渐变色彩

- **主渐变**: Primary400 → Accent400 - 用于主要按钮、重要元素
- **成功渐变**: Secondary400 → Secondary600 - 用于成功状态、确认按钮
- **警告渐变**: Warning400 → Warning600 - 用于警告状态、提醒按钮
- **错误渐变**: Error400 → Error600 - 用于错误状态、取消按钮

## 5. 色彩使用规范

### 主要元素
- **主要按钮**: 使用主色调（Primary400/500）或主渐变
- **次要按钮**: 使用中性色（Neutral200/300）
- **确认按钮**: 使用成功色（Secondary400/500）或成功渐变
- **取消按钮**: 使用错误色（Error400/500）或错误渐变
- **警告按钮**: 使用警告色（Warning400/500）或警告渐变

### 文本
- **主要文本**: 使用中性色（Neutral800/900）
- **次要文本**: 使用中性色（Neutral600/700）
- **提示文本**: 使用中性色（Neutral500）
- **禁用文本**: 使用中性色（Neutral400）
- **链接文本**: 使用主色调（Primary500）

### 背景和容器
- **页面背景**: 使用中性色（Neutral50）
- **卡片背景**: 使用白色或中性色（Neutral100）
- **容器背景**: 使用白色或中性色（Neutral100）
- **深色模式背景**: 使用中性色（Neutral900）
- **深色模式卡片**: 使用中性色（Neutral800）

### 边框和分隔线
- **边框**: 使用中性色（Neutral200/300）
- **分隔线**: 使用中性色（Neutral200）
- **深色模式边框**: 使用中性色（Neutral700/800）
- **深色模式分隔线**: 使用中性色（Neutral700）

### 状态指示
- **成功状态**: 使用成功色（Secondary400/500）
- **警告状态**: 使用警告色（Warning400/500）
- **错误状态**: 使用错误色（Error400/500）
- **信息状态**: 使用主色调（Primary400/500）

## 6. 可访问性

### 对比度
- **文本与背景**: 确保文本与背景的对比度至少为4.5:1
- **按钮文本与背景**: 确保按钮文本与背景的对比度至少为4.5:1
- **大文本**: 确保大文本（18pt以上）与背景的对比度至少为3:1

### 色彩一致性
- 同一类型的元素在所有界面中使用相同的颜色
- 遵循色彩使用规范，确保色彩的一致性
- 定期审查界面，确保色彩使用符合规范

## 7. 实现建议

### 在Compose中使用
```kotlin
// 主色调
MaterialTheme.colorScheme.primary

// 辅助色
MaterialTheme.colorScheme.secondary

// 强调色
MaterialTheme.colorScheme.tertiary

// 背景色
MaterialTheme.colorScheme.background

// 表面色
MaterialTheme.colorScheme.surface

// 文本色
MaterialTheme.colorScheme.onBackground
MaterialTheme.colorScheme.onSurface
```

### 渐变使用
```kotlin
// 主渐变
Brush.linearGradient(
    colors = listOf(GradientStart, GradientEnd),
    start = Offset(0f, 0f),
    end = Offset(1f, 1f)
)
```

## 8. 维护

- 定期更新色彩系统，确保与设计趋势保持一致
- 收集用户反馈，不断优化色彩方案
